PILOT BOOK — THE FAIRWAY GRAPH

Your engineering work is a graph. Chart it like one.

Everything Farled observes is connected: actors work in sessions, sessions touch files, plans reserve footprints, signals link colliding work, tasks come from incidents, PRs resolve tasks, outcomes close the loop. Today that lives in tidy relational tables. The Fairway Graph is the next chart sheet: the same metadata, projected into a queryable knowledge graph — so questions that need relationships, not rows, get real answers.

ROADMAP — this page describes where Farled Cloud is headed, not what ships today. The graph tier is designed on Neo4j; current pilots run on the shipped SQLite engine. Nothing here changes the privacy model: paths, identities, timestamps, counts — never code.

The chart datum

(:Actor {type: human|agent, vendor})-[:WORKED]->(:Session)-[:TOUCHED]->(:File)
(:Plan {status, confidence})-[:RESERVES]->(:Path)
(:Signal {type, severity})-[:BETWEEN]->(:Session)
(:Task)-[:FROM]->(:SourceSignal)   (:PR)-[:RESOLVES]->(:Task)
(:File)-[:GENERATED_FROM]->(:File)   // schema → client, proto → stubs

Every node and edge is metadata Farled already collects. The graph is a projection, not new surveillance.

How it stays honest

  • Advisory analytics, never the hot path. Collision detection stays deterministic and local — the graph enriches, it never gates.
  • One-way, rebuildable projection. The graph is derived from the same org-scoped event stream as everything else; losing it loses insight, never truth.
  • Method stated in every output — like the Harbor report's tiers, every graph insight names its algorithm and its limits.
  • Open core intact. The OSS edge stays a single SQLite binary. The graph is part of the paid memory — which is the point: the memory is a knowledge graph.

Why Neo4j

Because agents speak MCP, and so does Neo4j: their official MCP tooling lets an agent compose Farled's tools (“file a passage plan”) with graph queries (“what does the history say about this footprint?”) in a single session. And because the pairing is almost suspiciously fitting: farled is the Swedish word for a marked, navigable channel — and Neo4j's roots are Swedish too. Two navigational aids from the same waters: one for ships, one for data.
